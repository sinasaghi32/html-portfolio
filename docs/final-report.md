# 자판기 관리 프로그램 최종 완료 보고서

## 1. 과제 개요

- 과목명: JAVA 프로그래밍
- 목표: 자판기 관리 프로그램 개발
- 목적: GUI, 예외처리, 자료구조, 파일/DB, 멀티스레드, Socket 프로그래밍을 통합한 프로그래밍 능력 향상
- 구현 언어: Java
- 구현 환경: Linux/Unix, Windows, macOS에서 동작 가능한 Java Swing 애플리케이션
- 컴파일러: `javac`

## 2. 상위 설계서

### 2.1 시스템 구성

본 프로그램은 다음 세 구성요소로 설계하였다.

1. **Vending Client GUI**
   - 일반 판매 화면과 관리자 화면을 독립적으로 제공한다.
   - 판매 화면에서는 돈 투입, 음료 구매, 반환을 수행한다.
   - 관리자 화면에서는 매출/재고/화폐/수금/상품 변경/비밀번호 변경/서버 전송을 수행한다.

2. **Local Management Database**
   - 외부 라이브러리 없이 CSV/TXT 파일 기반으로 구성한 별도 관리 데이터 저장소이다.
   - 음료 상태, 판매 기록, 비밀번호, 재고 이벤트, 수금 기록을 저장한다.

3. **Socket Server**
   - 여러 자판기 클라이언트가 전송하는 판매/재고/변경 이벤트를 TCP Socket으로 수신한다.
   - 수신 이벤트를 서버 로그에 저장하고 저재고 이벤트는 콘솔 알림으로 출력한다.

### 2.2 주요 클래스

| 클래스 | 역할 |
| --- | --- |
| `VendingMachineApp` | 프로그램 진입점, 서버/셀프테스트/GUI 실행 모드 선택 |
| `VendingFrame` | Swing GUI, 판매 화면과 관리자 화면 분리 |
| `VendingMachine` | 자판기 도메인 로직, 돈/판매/재고/관리 기능 |
| `InventoryLinkedList` | 음료 재고 관리를 위한 직접 구현 Linked List |
| `ActionStack` | 최근 판매/관리 작업 기록 Stack |
| `EventQueue` | Socket 전송 대기 이벤트 Queue |
| `SalesTree` | 일별/월별 매출 집계를 위한 Binary Tree |
| `MoneyBox`, `MoneySession` | 투입금/거스름돈/수금 처리 |
| `SalesDatabase` | 파일 기반 관리 DB 읽기/쓰기 |
| `SocketSyncWorker` | 클라이언트 이벤트 전송 스레드 |
| `VendingServer` | TCP Socket 서버 |

### 2.3 화면 설계

- **판매 화면**
  - 음료 버튼 8개 표시
  - 금액 투입 버튼: 10원, 50원, 100원, 500원, 1,000원
  - 반환 버튼
  - 관리자 메뉴 진입 버튼
  - 판매 가능 여부에 따라 음료 버튼 활성/비활성

- **관리자 화면**
  - 관리자 비밀번호 인증 후 진입
  - 관리 화면이 활성화되면 판매 화면은 동작하지 않음
  - 재고 보충, 상품명/가격 변경, 수금, 서버 전송, 비밀번호 변경 가능

### 2.4 데이터 저장 설계

- `drinks.csv`: 음료 ID, 이름, 가격, 재고
- `sales.csv`: 판매 날짜, 자판기 ID, 음료 ID, 이름, 가격, 수량
- `password.txt`: 관리자 비밀번호
- `stock-events.csv`: 재고 보충/소진 관련 이벤트
- `collections.csv`: 관리자 수금 이력
- `events.log`: 서버가 수신한 Socket 이벤트

## 3. 상세 설계서

### 3.1 판매 처리 흐름

1. 사용자가 화폐 버튼을 누른다.
2. `MoneySession` 객체가 없으면 새로 생성한다.
3. 허용 화폐 단위인지 검사한다.
4. 지폐 투입액 5,000원 이하 및 총 투입액 7,000원 이하인지 검사한다.
5. 투입 금액과 재고, 거스름돈 가능 여부를 기준으로 구매 가능한 음료 버튼만 활성화한다.
6. 사용자가 음료를 선택하면 재고 1개 감소, 판매 기록 저장, 거스름돈 지급, Socket 이벤트 큐 등록을 수행한다.
7. 판매 완료 후 `MoneySession` 참조를 `null`로 바꾸어 세션 객체를 해제한다.

### 3.2 반환 처리 흐름

1. 사용자가 반환 버튼을 누른다.
2. 현재 투입 금액이 0이면 안내 메시지를 출력한다.
3. `MoneyBox`가 보유한 동전으로 거스름돈 조합을 계산한다.
4. 반환이 가능하면 동전 수량을 차감하고 반환 내역을 표시한다.
5. 반환 후 `MoneySession`을 해제한다.

### 3.3 관리자 처리 흐름

1. 관리자 메뉴 버튼 클릭
2. 비밀번호 인증
3. 인증 성공 시 `CardLayout`을 관리자 화면으로 전환
4. 관리자 화면 활성화 중 판매 기능은 사용할 수 없음
5. 관리자 기능 수행 시 DB 파일에 즉시 기록

### 3.4 Socket 처리 흐름

1. 판매, 저재고, 상품 변경, 스냅샷 이벤트를 `EventQueue`에 저장한다.
2. `SocketSyncWorker` 스레드가 Queue에서 이벤트를 꺼낸다.
3. `127.0.0.1:5050` 서버로 TCP Socket 연결 후 이벤트 문자열을 전송한다.
4. 서버는 클라이언트마다 별도 스레드를 생성하여 메시지를 처리한다.
5. 서버는 `data/server-db/events.log`에 이벤트를 누적 저장한다.

### 3.5 직접 구현 자료구조

- Linked List: 음료 목록 및 재고 관리
- Stack: 최근 판매/관리 작업 추적
- Queue: 서버 전송 이벤트 버퍼
- Tree: 판매 날짜 기준 매출 집계

### 3.6 Sort/Search 적용

- 삽입 정렬: 음료 ID 기준 정렬
- 선택 정렬: 관리자 가격 정렬 뷰 예시
- 선형 검색: Linked List에서 음료 ID 검색
- 이진 검색: ID 정렬 배열에서 음료 검색

### 3.7 예외 처리

- 허용되지 않는 화폐 단위
- 금액 상한 초과
- 재고 없음
- 거스름돈 부족
- 잘못된 관리자 비밀번호 규칙
- 파일 읽기/쓰기 실패
- Socket 연결 실패

## 5. 실행 및 시연 방법

1. `javac -encoding UTF-8 -d out src/vending/VendingMachineApp.java`로 컴파일한다.
2. 서버 터미널에서 `java -cp out vending.VendingMachineApp --server 5050`을 실행한다.
3. 클라이언트 터미널에서 `java -cp out vending.VendingMachineApp`을 실행한다.
4. 판매 화면에서 돈을 투입하고 활성화된 음료를 구매한다.
5. 관리자 메뉴에서 `Admin#123`으로 로그인한다.
6. 재고 보충, 상품명/가격 변경, 수금, 서버 전송 기능을 시연한다.
7. 서버 로그 `data/server-db/events.log`에서 수신 이벤트를 확인한다.

## 6. 추가 특장점

- Socket 서버가 꺼져 있어도 GUI는 계속 동작하며 전송 실패를 콘솔에 표시한다.
- 저재고 모니터가 10초 주기로 동작하여 재고 2개 이하 음료를 서버 이벤트로 보낸다.
- `--self-test` 실행 모드를 제공하여 핵심 판매/재고/비밀번호 기능을 자동 검증한다.
